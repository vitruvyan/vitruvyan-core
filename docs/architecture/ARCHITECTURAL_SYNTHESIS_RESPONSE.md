## 3️⃣ Doctrine Hardening

**Neural Engine Doctrine:**

The Neural Engine is Vitruvyan's quantitative nervous system. It evaluates entities through declared criteria, transforming domain data into standardized numerical scores without prediction, decision, or semantic meaning. It exists as pure computational substrate - a mathematical spreadsheet that computes formulas on provided data but contains no domain knowledge, business logic, or external dependencies. The Engine measures; verticals interpret. It is mandatory infrastructure, not optional tooling.

**Core ↔ Vertical Relationship Doctrine:**

Vitruvyan Core is intentionally incomplete without vertical incarnation. The Core enforces evaluation discipline through contract interfaces but never defines what matters. Vertical domains provide essential incarnation through concrete factors and profiles, completing the Core's computational skeleton with domain meaning. This relationship is not plugin-optional but architecturally mandatory - Core without verticals is bare metal; verticals without Core lack quantitative foundation. Incarnation happens at runtime through interface implementation, not configuration files.
